Tutorial

Creating a font with Calligro consists of generating a font template, drawing characters in it, and then exporting a bitmap font. The template can be filled by using a regular TTF font and/or drawing in any graphics editor.

To see examples on how to use an exported font with different game engines and frameworks check out the samples.

UI Overview

To create or open a project use the "File" menu. When a project is open the UI is divided into three sections:

  • Template Settings - defines the character set, size, and other properites impacting what you'll be drawing
  • Font Settings - defines other properties that can still be tweaked after drawing the characters
  • Font Preview - shows a preview of how the font will look like with the current settings

Template Settings

In the template settings you'll define the template, which is a PNG image with blank spots for you to draw your characters in.

You can always preview how the template will look like by pressing the "Preview Template" button at the bottom. For a freshly created project you'll see a regular grid of transparent squares. Each square corresponds to a different character, like "a", "b", "c", etc, for a total of 95 characters by default.

Pre-fill inputs

Close the preview and take a look at the "Prefill" input at the top of the template settings. It allows you to choose a regular TTF font to prefill the template with. Choose a font. You'll immediately see the preview on the right side of the program display sample text.

You can now also change the font fill color, outline color, and outline width with the inputs just below the prefill input. When you're done, preview the template again by pressing the "Preview Template" button. You'll see the template filled with the chosen font and colors.

Character set

Below the prefill input you'll find the "Characters set" button. Press it to open a dialog where you can choose the characters that will be included in the template. You can feely write any characters you'd like. You can also start from a preset like "Basic Latin" or "Hiragana" using the "Override with preset" button.

Character size and base

Below the character set input you'll find the width, height and base inputs. The width and height define the size of each character in pixels.

The base defines the height from the top of a character at which the character "sits". Everything below the base will stick out like in the letters "j" and "g". For example, a character of height 100 and base 70 will have 30 pixels of space below the base.

When you preview the template again you'll see horizontal lines between character slots. Those lines represent the base of each character.

Size overrides

Below the size and base inputs you'll find the "Size overrides" control. It allows you to define different sizes for different characters.

Press the "+" button to open a dialog where you can add characters to the list of size overrides. For example, write "abc" in the dialog's input and press "Add all" to add the characters "a", "b" and "c".

Exporting and importing templates

Before exporting the template you can also play with the Font Settings section (explained later) and preview how the font will look like. Font settings have no impact on the exported template. If you're happy with how the font looks like just by using the prefill inputs, you can skip exporting the template. If you want to draw your own characters, you need to export the template.

Exporting template

To export the template, you can either:

  • press "Preview Template" and then "Export PNG", or
  • choose the "File -> Export Template PNG" menu at the top
In the desktop app and in the web app on modern browsers you'll be prompted to choose a location to save the template. In the web app on browsers without the support of the File System API you'll be prompted to download the template file.

You should now have a PNG file you can edit in any graphics editor. Remember that the horizontal lines between characters represent the character base. After you're done drawing your characters, you can import the PNG back into Calligro.

Importing templates

Import a filled template back to calligro by choosing "File -> Import Template PNG" from the menu at the top. You'll immediately see your custom characters in the font preview on the right.

Font Settings

In Font Settings you can define additional properties, like spacing, line height, and kerning pairs. Unlike Template Settings, those properties can still be changed after the template is imported.

Spacing and line height

The horizontal and vertical spacing is the distance between characters in pixels. If the spacing is set to 0, character boundaries (as defined by the template) will touch each other.

Line height is the vertical distance between lines of text. The actual distance between lines will be the line height plus the vertical spacing.

Kerning pairs

Kerning pairs allow you to define custom spacing between specific characters. For example, if you want the letter "a" to be closer to "b" than to "c", you can add a kerning pair for "ab" with a negative distance. The distance is the number of pixels by which the second character in the pair will be moved further or closer to the first one.

Once a pair is added, you can edit it by choosing it from the dropdown next to the "+" button and modifying the distance input. You can also remove it by pressing the "-" button next to the dropdown.

Exporting a font

Once you're happy with the font settings, you can export the font by pressing the "File -> Export Font" menu at the top. You can choose between two formats: TXT and XML. Some game engines and frameworks require the TXT format and some require the XML format. Unfortunately, this is usually not documented anywhere, so you might need to try both. Some known examples include:

Tool Format
Godot TXT
LibGDX TXT
LÖVE TXT
Heaps.io TXT
Phaser XML
HaxeFlixel XML

To see examples on how to use an exported font with different game engines and frameworks check out the samples.
